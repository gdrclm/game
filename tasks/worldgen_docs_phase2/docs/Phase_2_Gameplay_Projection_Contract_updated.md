# Phase_2_Gameplay_Projection_Contract
## Official bridge contract from Phase 2 environmental logic to gameplay-facing meaning
**Repository:** `gdrclm/game`  
**Status:** Updated draft source-of-truth contract after completed Phase 1  
**Audience:** design, engineering, Codex implementation passes  
**Scope:** defines how `PressureFieldPackage` and `EnvironmentalRhythmPackage` must be translated into gameplay-facing signals after completed Phase 1

---

# 1. Purpose

This contract defines the official gameplay projection contract for Phase 2 after completed Phase 1.

Its purpose is to prevent:
- Phase 2 becoming deep but gameplay-detached simulation;
- environmental pressure existing only as abstract math;
- runtime systems inventing their own travel / shelter / recovery truth;
- island progression flattening into generic “later islands are harder”.

---

# 2. Upstream basis

Phase 2 gameplay projection now stands on top of completed Phase 1 exports:
- `MacroGeographyPackage`
- filtered, explicitly allowed parts of `MacroGeographyHandoffPackage`

That means gameplay projection must reflect:
- official physical truth from root-package records;
- official structural truth from root-package records;
- environmental interpretation generated by Phase 2;
- record-bound environmental profiles produced by Phase 2.

It must not:
- bypass completed Phase 1 contracts;
- invent gameplay meaning unrelated to Phase 2 packages.

---

# 3. Core principle

Phase 2 does **not** directly generate:
- island stories;
- ideologies;
- factions;
- final layouts;
- final spawn placements.

Phase 2 generates:
- environmental burden
- environmental timing
- environmental recoverability

Gameplay projection translates those into:
- traversal expectations
- recovery expectations
- shelter reliability expectations
- scarcity expectations
- hazard expectations
- route-window expectations
- environmental planning styles

---

# 4. Mandatory projection layers

## Layer A — Traversal Layer
- traversal pressure
- terrain penalty expectation
- route fragility expectation
- route window expectation
- chokepoint traversal stress
- detour pressure

## Layer B — Survival Layer
- food reliability expectation
- water reliability expectation
- shelter reliability expectation
- recovery tempo expectation
- exposure penalty expectation
- scarcity pressure expectation

## Layer C — Hazard Layer
- hazard intensity band
- hazard volatility band
- catastrophe likelihood band
- weather aggression expectation
- disruption cadence expectation
- rupture expectation

## Layer D — Relief Layer
- safe window expectation
- stable-zone expectation
- recovery window expectation
- resource pocket likelihood
- temporary calm expectation
- navigable interval expectation

### Hard rule
If Layer D is absent, gameplay projection is incomplete.

---

# 5. Record-aware gameplay projection rule

Because completed Phase 1 now exports rich records and Phase 2 must bind to those records, gameplay projection must also be able to consume:
- record-bound environmental summaries
- record-bound profile families
- region-specific burden/timing meaning

Gameplay projection must not operate only on anonymous global averages.

# 7. Pressure vs rhythm mapping rule

## Pressure may influence:
- magnitude
- burden
- persistence
- cost
- reliability loss
- danger severity

## Rhythm may influence:
- cadence
- timing
- windows
- predictability
- rupture frequency
- recovery tempo
- alternation between strain and relief

### Hard prohibition
Rhythm must not be flattened into pure intensity.  
Pressure must not substitute for timing structure.

# 9. Required runtime-facing meanings

Projection must support later mapping into:
- expedition progression
- island layout bias
- world spawn weighting
- map/debug summaries

It must be able to say:
- what kind of run this is;
- what the player should fear;
- what the player may still rely on;
- whether timing matters more than endurance;
- whether recovery is rare but powerful or common but weak.

---

# 10. Mandatory relief rule

This remains the highest-priority design rule:

Phase 2 gameplay projection must not project only punishment.

It must explicitly export:
- recovery windows
- safe windows
- stable zones
- temporary calm
- resource pockets
- navigable intervals

Without that, the player's planning space collapses.

---

# 11. Acceptance criteria

This contract is satisfied only if:
- Phase 2 outputs map into stable gameplay-facing axes;
- timing meaning survives into projection;
- recovery remains explicit;
- record-bound profiles remain available;
- progression/layout/spawn/map layers can later consume the projection without inventing new environmental truth.

---

# 12. Final statement

Gameplay projection exists so Phase 2 does not merely say:
- the world is harsh

It must say:
- how harshness changes movement,
- how timing changes planning,
- where relief can exist,
- what kind of run a region implies,
- and what the player can still rely on.
