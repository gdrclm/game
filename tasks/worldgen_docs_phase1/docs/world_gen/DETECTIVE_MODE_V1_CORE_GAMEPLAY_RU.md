# DETECTIVE MODE V1 CORE GAMEPLAY

## Статус

Source-of-truth документ для V1 детективного режима как одного из столпов gameplay.

Этот документ исправляет прежнюю трактовку, где детективность описывалась только как narrative/diagnosis layer через письма, дневник, осмотр и debug-носители.

В V1 детективный режим является отдельной gameplay-системой, доступной с начала игры.

---

# 1. Главная формула

```text
Детективный режим = активируемая игроком gameplay-система,
которая помогает в рамках выбранного квеста найти конкретные точки прорыва расследования.
```

Игрок не просто читает текст.
Игрок действует как сыщик:

```text
осматривает объект;
связывает улики;
допрашивает свидетеля;
находит противоречие;
собирает гипотезу;
сверяет улику с заметкой друга;
проверяет версию через действие в мире.
```

---

# 2. Герой и друг: интеллектуальное разделение ролей

| Герой | Друг |
|---|---|
| читает следы | читает симптомы |
| допрашивает людей | анализирует образцы |
| видит мотивы | видит патологию |
| понимает ложь | понимает тело |
| собирает причинность | собирает формулу |
| проверяет версию в мире | уточняет медицинскую гипотезу |

Герой не заменяет друга.
Друг не заменяет героя.

V1 должна показать игроку, что расследование работает только при связке:

```text
следы мира + показания людей + документы + письмо друга + медицинская гипотеза.
```

---

# 3. Доступность с начала игры

Детективные механики доступны с самого начала:

```text
осмотр;
реконструкция;
допрос;
сопоставление фактов;
связывание улик;
доска причинности;
проверка гипотез;
сверка с письмом друга.
```

Survival-механики при этом тяжёлые с начала:

```text
мало слотов;
быстрая усталость;
плохой сон вне безопасных мест;
ошибки в ремесле;
низкая эффективность сбора;
незнание местных ресурсов;
дорогие ошибки в приготовлении еды;
плохая переносимость голода.
```

Итоговый опыт:

```text
герой силён как сыщик,
но слаб как выживальщик.
```

# 4. Основные механики детективного режима V1

## 4.1. Механика 1 — Осмотр

Игрок осматривает объект, место или след.

Пример:

```text
Факт:
ящик открыт не ломом, а штатным ключом.

Вывод:
взлома не было.

Вопрос:
кто имел доступ?
```

В V1 осмотр должен давать не просто описание, а структурированный результат:

```text
факт;
вывод героя;
новый вопрос;
связанные улики;
возможная связь с письмом друга;
потенциальное противоречие.
```

## 4.2. Механика 2 — Противоречие

NPC говорит одно, мир показывает другое.

Пример:

```text
NPC:
“Я не заходил в хранилище.”

Герой видит:
на его рукаве пыль от архивной плесени.

Вывод:
NPC был рядом с архивом или хранилищем.
```

В V1 противоречие возникает между:

```text
репликой NPC;
осмотренным объектом;
документом;
письмом друга;
физическим следом;
историей острова;
медицинским симптомом.
```

## 4.3. Механика 3 — Связь улик

Игрок связывает несколько объектов в причинную цепочку.

Пример:

```text
ключ → хранилище → мокрая ткань → лабораторный образец → письмо друга
```

Связь улик должна быть gameplay-действием, а не только текстовым выводом.

Минимум V1:

```text
игрок может выбрать улику;
игрок может связать её с другой уликой/заметкой;
система фиксирует связку;
связка может открыть гипотезу, вопрос или прорыв расследования.
```

## 4.4. Механика 4 — Гипотеза

Игрок выбирает не “кто убийца”, а:

```text
Что произошло?
Почему это произошло?
Что все неправильно поняли?
Какая причина глубже видимого симптома?
Что нужно проверить дальше?
```

Гипотеза в V1 должна иметь статус:

```text
неполная;
подтверждённая частично;
противоречивая;
ложная;
рабочая;
достаточно подтверждённая для действия.
```

## 4.5. Механика 5 — Сверка с письмом друга

Письма друга не просто дают квест.
Они являются интеллектуальным инструментом.

Игрок сверяет улику с:

```text
симптомом из письма;
лабораторной гипотезой друга;
ошибочной версией друга;
новым вопросом друга;
требуемым компонентом лекарства;
предыдущим островным выводом.
```

# 5. Кнопка детективного режима

В V1 вводится отдельная кнопка:

```text
Detective Mode / Режим сыщика / Детективный режим
```

Функция:

```text
показать игроку, где в рамках выбранного текущего квеста он может получить прорыв расследования.
```

Важно:
Это не глобальный wallhack.
Это не подсветка всех интерактивных объектов.
Это не подсказка решения.

Режим работает только в контексте:

```text
выбранного активного квеста;
текущей стадии расследования;
уже известных игроку фактов;
доступных действий на текущем экране.
```

# 6. Quest-scoped подсветка на игровом экране

## 6.1. Как активируется

Игрок:

```text
1. выбирает активный квест / цель письма друга / расследовательную задачу;
2. переходит на экран gameplay;
3. нажимает кнопку детективного режима;
4. система подсвечивает только те места, где в рамках текущего квеста возможен прорыв расследования.
```

## 6.2. Что подсвечивается

Только объекты с ролью:

```text
breakthroughObject;
contradictionObject;
evidenceCarrier;
friendNoteRelevantObject;
hypothesisCheckObject;
witnessTriggerObject;
reconstructionAnchor;
```

Примеры:

```text
дверь хранилища;
ящик, открытый штатным ключом;
мокрая ткань;
архивная полка;
соляная бочка;
сломанный фильтр;
следы у причала;
рабочее место свидетеля;
доска объявлений с запретом;
объект, который сверяется с письмом друга.
```

## 6.3. Как выглядит подсветка

V1 visual direction:

```text
тонкий контур;
мягкое пульсирование;
ограниченная яркость;
без fantasy glow;
без огромных стрелок;
без floating quest marker.
```

Подсветка должна выглядеть как фокус внимания сыщика, а не магическая подсказка.

Возможный стиль:

```text
тонкая тёмно-светлая обводка;
короткая бумажная пометка в стиле блокнота;
минимальный eye-symbol рядом с объектом, если UI это поддержит;
подсветка только после нажатия кнопки режима.
```

## 6.4. Условия появления

Подсветка появляется, если:

```text
активен детективный режим;
выбран конкретный расследовательный квест;
объект релевантен текущей стадии квеста;
игрок уже имеет минимальный контекст, чтобы объект был честной подсказкой;
объект находится на текущем экране gameplay;
объект ведёт к реальному прорыву, новой улике, противоречию или проверке гипотезы.
```

Подсветка НЕ появляется, если:

```text
нет выбранного квеста;
объект просто интерактивен, но не важен для текущего расследования;
объект относится к другой ветке;
объект ещё не должен быть понятен игроку;
подсветка раскрыла бы финальное решение слишком рано;
объект является обычным ресурсом survival.
```

# 7. Dialogue-scoped подсветка

Детективный режим работает также внутри диалога.

Игрок:

```text
1. говорит с NPC;
2. выбирает активный квест / цель;
3. нажимает кнопку детективного режима или включает его заранее;
4. нужный вариант расспроса подсвечивается, если он ведёт к прорыву расследования.
```

## Формат диалога V1

Диалоги должны быть короткими и функциональными.

Базовые темы:

```text
[болезнь]
[склад]
[друг]
[местные слухи]
[нужный ресурс]
```

## Значки тем

| Значок | Значение |
|---|---|
| глаз | тема содержит наблюдение |
| восклицание | есть противоречие |
| письмо | связано с другом |
| колба | связано с лекарством |
| монета | связано с экономикой |
| маска | NPC может лгать |

Важно:
Значки не обязаны раскрывать правду.
Они показывают тип интеллектуальной ценности темы.

## Что подсвечивается в диалоге

Только варианты, которые:

```text
открывают новую улику;
проверяют противоречие;
связаны с письмом друга;
дают medical/diagnosis progress;
открывают witness lead;
запускают реконструкцию;
проверяют текущую гипотезу.
```

Не подсвечиваются:

```text
обычная болтовня;
уже исчерпанная тема;
ресурсная торговля без detective value;
общий lore без связи с активным квестом;
вариант, который ведёт только к survival, но не к расследованию.
```

# 8. Функция “Что здесь важно?”

В V1 детективный режим должен отвечать на вопрос игрока:

```text
Что здесь важно для текущего расследования?
```

Это не кнопка “решить загадку”.
Это фильтр внимания.

Функция показывает:

```text
куда смотреть;
кого спросить;
какой объект связан с текущим вопросом;
какая тема диалога может дать прорыв;
какую улику стоит сверить с письмом друга.
```

Функция не показывает:

```text
правильную гипотезу;
финальный ответ;
все будущие улики;
объекты из других квестов;
моральные последствия заранее;
полный маршрут прохождения.
```

# 9. Структура detective quest V1

Каждый detective quest должен иметь:

```text
questId;
activeQuestion;
knownFacts[];
evidenceSlots[];
contradictionSlots[];
witnessTopics[];
friendNoteRefs[];
hypothesisOptions[];
breakthroughTargets[];
reconstructionAnchors[];
requiredSurvivalConflict;
progressState;
```

## progressState

```text
not_started;
observed_first_fact;
contradiction_found;
evidence_linked;
hypothesis_opened;
hypothesis_tested;
breakthrough_reached;
resolved_partial;
resolved_strong;
false_resolution;
```

---

# 10. Объекты детективного режима

Каждый объект, который может подсвечиваться в detective mode, должен иметь explicit role.

```js
DetectiveObjectRole = {
  inspectable: true,
  breakthroughObject: false,
  evidenceCarrier: false,
  contradictionObject: false,
  reconstructionAnchor: false,
  friendNoteRelevantObject: false,
  hypothesisCheckObject: false,
  survivalConflictObject: false,
  activeQuestRefs: [],
  requiredKnownFactRefs: [],
  unlocks: []
}
```

Объект не подсвечивается просто потому, что он интерактивный.
Он подсвечивается только из-за связи с активным detective quest.

# 11. Диалоговые темы детективного режима

Каждая тема диалога должна иметь metadata:

```js
DialogueInvestigationTopic = {
  topicId,
  label,
  npcId,
  activeQuestRefs: [],
  iconTypes: [],
  breakthroughValue: false,
  contradictionValue: false,
  friendNoteValue: false,
  medicineValue: false,
  economyValue: false,
  deceptionRisk: false,
  requiredKnownFactRefs: [],
  unlocksEvidenceRefs: [],
  unlocksContradictionRefs: [],
  unlocksHypothesisRefs: []
}
```

Иконки:

```text
eye = observation;
exclamation = contradiction;
letter = friend note;
flask = medicine;
coin = economy;
mask = deception risk.
```

# 12. Доска причинности V1

Доска причинности в V1 — это не обязательно большой финальный UI.
Но минимальная система должна существовать.

Минимум:

```text
список найденных фактов;
список улик;
список заметок друга;
список противоречий;
список открытых гипотез;
связи между ними;
статус гипотезы.
```

Доска отвечает на вопросы:

```text
что известно;
что противоречит;
что связано с письмом друга;
какая версия открыта;
чего не хватает для проверки;
какая улика дала прорыв.
```

# 13. Первый 30-минутный учебный блок

После первых 30 минут игрок должен понимать:

```text
я играю за профессионального сыщика;
друг важен не только сюжетно, но и интеллектуально;
механики расследования — основа игры;
первая версия событий почти всегда ложная;
мир устроен причинно-следственно;
те же навыки нужно применять к островам, поселениям и болезни;
survival будет мешать расследованию;
голод, сон, опасность и чужая культура меняют цену правильного вывода.
```

## Учебный сценарий должен показать 4 механики

1. Осмотр:

```text
ящик открыт штатным ключом, а не взломан.
```

2. Противоречие:

```text
NPC отрицает вход в хранилище,
но на рукаве архивная плесень.
```

3. Связь улик:

```text
ключ → хранилище → мокрая ткань → образец → письмо друга.
```

4. Гипотеза:

```text
не “кто виноват”,
а “что произошло, почему, и что все неправильно поняли”.
```

# 14. Связь с островным gameplay

То, чему игрок учится в первом блоке, затем применяется на масштабе островов:

```text
место происшествия -> остров;
ящик/хранилище -> склад/фильтр/порт/колодец;
один NPC -> поселение/фракция;
мокрая ткань -> цепочка хранения/воды/соли;
письмо друга -> медицинская гипотеза;
локальная ложь -> ложная причинность архипелага;
частный мотив -> экономика/политика/религия;
локальная реконструкция -> systemic diagnosis.
```

# 15. Связь с четырьмя итоговыми шкалами

Детективный режим влияет на:

## DiagnosisCompleteness

Растёт, когда игрок:

```text
находит улики;
связывает улики;
выявляет противоречия;
собирает рабочую гипотезу;
отбрасывает ложную причинность.
```

## MedicalReadiness

Растёт, когда игрок:

```text
находит образцы;
сверяет их с письмом друга;
добывает компоненты;
проверяет медицинские гипотезы;
открывает новые данные для формулы.
```

## SocialTrust

Меняется, когда игрок:

```text
допрашивает NPC;
ловит NPC на лжи;
идёт прямым конфликтом;
находит мягкий третий путь;
выбирает, кому раскрыть правду.
```

## SystemicRepair

Растёт, когда игрок:

```text
понимает не симптом, а источник;
находит repair target;
получает поддержку или ресурсы;
чинит фильтр/маршрут/склад/источник/инфраструктурную ошибку.
```

---

# 16. V1 scope

В V1 должно быть реализовано:

```text
активный detective mode button;
quest-scoped подсветка breakthrough targets на gameplay screen;
dialogue-scoped подсветка релевантной темы расспроса;
иконки тем диалога;
осмотр с факт/вывод/вопрос структурой;
противоречия между NPC/объектами/заметками;
связывание улик;
минимальная доска причинности;
гипотезы со статусами;
сверка улик с письмом друга;
первый 30-минутный tutorial block;
связь detective progress с DiagnosisCompleteness / MedicalReadiness / SocialTrust / SystemicRepair.
```

---

# 17. Что не входит в V1

```text
полный Phoenix Wright-style court system;
полная voiced dialogue system;
огромные dialogue trees;
автоматическое раскрытие правильного ответа;
полная кинематографичная реконструкция событий;
полный UI всех связей на 30 островов;
сложная reputation simulation;
полная NPC schedule simulation;
автоматическое подсвечивание всех интерактивных объектов;
подсветка объектов без активного квеста;
подсветка финального решения раньше нужной стадии.
```

---

# 18. Acceptance criteria

V1 detective mode принимается, если:

```text
игрок может включить detective mode;
режим работает только в рамках активного квеста;
на gameplay screen подсвечиваются только breakthrough-relevant объекты;
в диалоге подсвечиваются только breakthrough-relevant темы;
значки тем передают тип ценности: observation/contradiction/friend/medicine/economy/deception;
осмотр выдаёт факт, вывод и вопрос;
NPC может создать противоречие с уликой;
игрок может связать минимум две улики и одну заметку друга;
игрок может открыть и проверить гипотезу;
первый tutorial за 30 минут обучает четырём механикам;
detective progress влияет на четыре итоговые шкалы;
режим не показывает финальный ответ;
режим не подсвечивает всё подряд.
```

---

# 19. Final statement

Детективный режим V1 — это не декоративный слой текста.

Это gameplay pillar, который учит игрока:

```text
смотреть;
сравнивать;
сомневаться;
допрашивать;
связывать;
строить гипотезу;
проверять её действием;
отличать симптом от причины.
```

И затем переносит этот навык с маленькой сцены на весь архипелаг.
