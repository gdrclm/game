WORLDGEN_V1_PHASE_ASSET_CATALOG_RU_DETAILED.txt

Статус:
Детальный каталог визуальных ассетов V1 по фазам 0–26.

Назначение:
Этот документ НЕ является обзором фаз.
Это каталог конкретных визуальных ассетов, тайлов, объектов, prop-clusters, debug-панелей и placeholder-носителей, необходимых для визуального тестирования каждой фазы V1.

Каждая запись должна отвечать:
- что это;
- тип ассета;
- где используется;
- как выглядит;
- какую роль играет;
- когда появляется;
- из какой фазы получает причину;
- в какой фазе материализуется;
- чего НЕ делает.

Жёсткое правило:
Phase 0–2 не создают человеческую инфраструктуру. Они создают физическую основу и debug/terrain-предпосылки.
Phase 18–26 закрывают материализацию визуальных ассетов через wrappers.

Типы ассетов:
DEBUG_PANEL — таблица/карточка/отчёт для разработчика.
DEBUG_OVERLAY — отладочный слой карты, heatmap, линии, зоны.
MAP_SYMBOL — символ на debug/manual map, не игровой floating marker.
GAME_TILE — изометрический тайл поверхности.
EDGE_TILE — край/переход тайла.
WORLD_OBJECT — физический объект в мире.
PROP_CLUSTER — группа props, читаемых как один смысловой объект.
NPC_PLACEHOLDER — временный NPC-спрайт/силуэт для теста.
TEXT_CARRIER — письмо друга, дневник, заметка, текст осмотра, диалог.
VALIDATION_REPORT — отчёт Phase 26.
V2_SLOT — зарезервировано под V2, не требуется как готовый арт в V1.

PHASE 0 — MASTER SEED: ASSET PACK
============================================================

PHASE0_ASSET_001 — WORLD_PROFILE_PANEL
Тип: DEBUG_PANEL
Где используется: manual test export, seed comparison export.
Что это: карточка профиля мира.
Как выглядит: прямоугольная таблица с крупными строками: maritimeDependence, routeFragility, environmentalVolatility, collapseIntensity, conflictPressure, culturalPermeability, archipelagoPotential.
Роль: позволяет увидеть, какие базовые тенденции seed задаёт всему миру.
Когда появляется: всегда при генерации V1.
Причина: Phase 0 master seed.
Материализация: debug export, не игровой мир.
Не делает: не создаёт тайлы, острова, здания, NPC.

PHASE0_ASSET_002 — SEED_AXIS_ICON_ROW
Тип: DEBUG_PANEL / MAP_SYMBOL
Где используется: WorldProfilePanel, three-seed comparison.
Что это: строка маленьких условных символов для осей seed.
Как выглядит: простые пиктограммы: волна для maritime, треснувшая линия для routeFragility, молния/облако для volatility, обрыв/руина для collapse.
Роль: ускоряет ручное сравнение seed-профилей.
Когда появляется: всегда в debug.
Причина: Phase 0 tendencies.
Материализация: debug only.
Не делает: не является игровым UI и не появляется над картой.

PHASE0_ASSET_003 — SUBSEED_NAMESPACE_TABLE
Тип: DEBUG_PANEL
Где используется: debug export.
Что это: таблица phaseId / namespace / seedFragment.
Как выглядит: техническая таблица с phase rows 0–26.
Роль: проверяет детерминированность генерации.
Когда появляется: всегда.
Причина: Phase 0 sub-seed allocation.
Материализация: debug export.
Не делает: не влияет визуально на gameplay.

PHASE0_ASSET_004 — SEED_TO_PHASE_INFLUENCE_ARROWS
Тип: DEBUG_PANEL / DEBUG_GRAPH
Где используется: manual visual test.
Что это: схема влияния Phase 0 на поздние фазы.
Как выглядит: стрелки от seed axis к Phase 1, 2, 11, 13, 15, 17.5.
Роль: показывает причинное наследование.
Когда появляется: всегда в readiness/export.
Причина: Phase 0 causal contract.
Материализация: debug graph.
Не делает: не создаёт конкретные ассеты мира.
